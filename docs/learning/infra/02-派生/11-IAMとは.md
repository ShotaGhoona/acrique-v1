# IAMとは - なぜ専用ユーザーを作るのか

**読了時間: 1分**

---

## 一言で言うと

**IAM = AWSの「社員証発行システム」**

---

## 日常例で理解

```
会社のビルに例えると：

ルートユーザー = 社長のマスターキー
  → 全部屋に入れる
  → 金庫も開けられる
  → 紛失したら会社終わる

IAMユーザー = 社員ごとのカードキー
  → 必要な部屋だけ入れる
  → 権限を細かく設定できる
  → 漏れても被害を限定できる
```

---

## 今やろうとしていること

```mermaid
graph LR
    Root[ルートユーザー<br/>今ログイン中] -->|作成| IAM[IAMユーザー<br/>acrique-deploy]
    IAM -->|使う| CLI[AWS CLI<br/>ターミナルから操作]
    CLI -->|デプロイ| AWS[AWSリソース作成]
```

**ルートユーザーでCLI操作 = 社長のマスターキーを持ち歩くようなもの**
→ 危険なので、専用の「デプロイ担当ユーザー」を作る

---

## やる / やらない の違い

| | IAMユーザーを作る | ルートユーザーのまま |
|--|------------------|-------------------|
| セキュリティ | 安全 | 危険 |
| キー漏洩時 | そのユーザーだけ削除 | アカウント全体が危険 |
| AWS推奨 | **これが正解** | 非推奨 |
| 実務 | 必須 | ありえない |

---

## 次にやること

1. **IAMユーザー作成** → デプロイ専用ユーザー
2. **アクセスキー発行** → CLIからログインするための鍵
3. **AWS CLI設定** → ターミナルからAWSを操作可能に

---

理解したら、IAMユーザー作成に進もう。
