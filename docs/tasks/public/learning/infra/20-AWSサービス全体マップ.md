# 21: AWSサービス全体マップ

## フルスタックエンジニアが知るべきサービス

### 必須レベル（これは絶対知っておく）

| カテゴリ | サービス | 用途 | ACRIQUEでの使用 |
|---------|---------|------|----------------|
| コンピュート | **EC2** | 仮想サーバー | ❌（ECS使用） |
| コンピュート | **ECS/Fargate** | コンテナ実行 | ✅ バックエンド |
| コンピュート | **Lambda** | サーバーレス関数 | ❌ |
| データベース | **RDS** | リレーショナルDB | ✅ PostgreSQL |
| データベース | **DynamoDB** | NoSQL | ❌ |
| ストレージ | **S3** | オブジェクトストレージ | ✅ 商品画像 |
| ネットワーク | **VPC** | 仮想ネットワーク | ✅ 基盤 |
| ネットワーク | **ALB** | ロードバランサー | ✅ 振り分け |

---

### 知っておくと良いレベル

| カテゴリ | サービス | 用途 | 一言メモ |
|---------|---------|------|---------|
| CDN | **CloudFront** | コンテンツ配信 | S3の前に置くと爆速 |
| DNS | **Route 53** | ドメイン管理 | 独自ドメイン使うなら必須 |
| 認証 | **Cognito** | ユーザー認証 | ログイン機能を秒で実装 |
| API | **API Gateway** | API管理 | Lambda使うなら必須 |
| 監視 | **CloudWatch** | ログ・メトリクス | 必ず使う |
| 秘密管理 | **Secrets Manager** | パスワード管理 | DB認証情報など |
| IaC | **CloudFormation** | インフラ定義 | CDKの裏側 |
| CI/CD | **CodePipeline** | デプロイ自動化 | GitHub Actionsの代替 |

---

### 名前だけ知っておくレベル

| カテゴリ | サービス | 用途 |
|---------|---------|------|
| コンテナ | EKS | Kubernetes管理 |
| キュー | SQS | メッセージキュー |
| 通知 | SNS | プッシュ通知 |
| メール | SES | メール送信 |
| 検索 | OpenSearch | 全文検索 |
| キャッシュ | ElastiCache | Redis/Memcached |
| WAF | WAF | ファイアウォール |
| BI | QuickSight | ダッシュボード |
| ML | SageMaker | 機械学習 |
| IoT | IoT Core | IoTデバイス管理 |

## 学習の優先順位

```
Level 1 (今日学んだ):
  VPC → EC2/ECS → RDS → S3 → ALB

Level 2 (次に学ぶ):
  CloudFront → Route53 → Cognito → Lambda → API Gateway

Level 3 (必要になったら):
  SQS → SNS → ElastiCache → OpenSearch

Level 4 (特殊用途):
  EKS → SageMaker → IoT
```
